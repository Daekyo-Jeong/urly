# Catalog — Design Brief

## 프로젝트 개요

URL을 등록하면 독립적인 macOS 앱(.app)을 생성해주는 데스크톱 유틸리티.
디자인 대상은 **카탈로그 관리 앱(메인 앱)** 1개와 **생성된 앱(SSB)의 최소 브라우저 크롬** 1개, 총 2개의 UI다.

---

## 디자인 대상

### 1. 카탈로그 관리 앱 (메인 앱)

사용자가 앱을 등록·수정·삭제·실행하는 관리 화면.

#### 화면 목록

| 화면 | 설명 |
|------|------|
| **앱 목록 (홈)** | 등록된 앱들을 그리드 또는 리스트로 표시. 각 항목에 아이콘, 이름, URL 표시. 편집/삭제/실행 액션 |
| **앱 등록** | URL 입력 필드, 앱 이름 입력, 아이콘 미리보기(자동 추출 결과) + 수동 업로드 버튼, 생성 버튼 |
| **앱 수정** | 등록 화면과 동일 구조. 이름·URL·아이콘 변경 가능 |
| **앱 삭제 확인** | 삭제 시 사용자 데이터(쿠키/세션) 함께 삭제 여부를 묻는 확인 다이얼로그 |

#### 인터랙션 요구사항

- 앱 목록에서 더블클릭 또는 실행 버튼 → 해당 앱 즉시 실행
- 앱 등록 시 URL 입력 후 포커스 아웃 → 자동으로 사이트 title + 아이콘 추출하여 미리보기
- 아이콘 영역 클릭 → 이미지 파일 선택 다이얼로그 (수동 지정)
- 아이콘은 원형 또는 macOS 앱 아이콘 형태(squircle)로 표시
- 빈 상태(앱 0개) → 비어있음을 알리고 등록 유도하는 empty state

#### 상태 표현

- URL에서 아이콘/제목 추출 중 → 로딩 인디케이터
- 추출 실패 → 기본 placeholder 아이콘 + 수동 설정 유도 메시지
- 앱 생성 중 → 진행 상태 표시 (stub .app 파일 생성 과정)
- 앱 생성 완료 → 성공 피드백

### 2. 생성된 앱 (Site-Specific Browser) 의 브라우저 크롬

각 생성된 앱이 실행될 때의 창 UI. 최소한의 브라우저 크롬만 존재.

#### 구성 요소

| 요소 | 설명 |
|------|------|
| **타이틀바** | macOS 네이티브 타이틀바. 현재 페이지 제목 표시. 신호등 버튼(닫기/최소화/최대화) |
| **다운로드 표시** | 파일 다운로드 시 진행률 표시. 위치: 하단 바 또는 토스트 |
#### 특이사항

- 주소창, 탭바, 북마크바, 뒤로/앞으로 버튼 없음
- 웹 콘텐츠가 창 전체를 차지해야 함 (타이틀바 제외)
- 다운로드는 시스템 기본 동작에 위임 (별도 UI 없음)
- macOS 네이티브 앱처럼 느껴져야 함

---

## 디자인 원칙

1. **macOS 네이티브 느낌**: 시스템 폰트(SF Pro), 네이티브 컨트롤 스타일, vibrancy/blur 등 macOS 디자인 언어 준수
2. **유틸리티 우선**: 장식 최소화. 기능에 집중하는 도구형 UI
3. **최소 인지 부하**: 앱 관리는 부가 작업이므로 빠르게 끝낼 수 있어야 함

---

## 디자인 시스템 요구사항

### 컬러

- macOS 시스템 컬러 기반 (NSColor semantic colors 대응)
- 다크모드 / 라이트모드 대응 필수
- Accent color: 시스템 설정 따르거나 앱 고유 1색 지정

### 타이포그래피

- SF Pro (macOS 시스템 폰트) 사용
- 크기 체계: macOS HIG 기준 (13pt body, 11pt caption, 20pt+ title 등)

### 컴포넌트 목록

디자인 시스템에 포함되어야 할 컴포넌트:

| 컴포넌트 | 용도 |
|----------|------|
| App Card | 앱 목록에서 각 앱을 표시하는 카드. 아이콘 + 이름 + URL + 액션 |
| Input Field | URL 입력, 앱 이름 입력 |
| Button | Primary(생성/저장), Secondary(취소), Danger(삭제) |
| Icon Preview | 아이콘 미리보기 + 교체 가능한 영역 |
| Dialog / Modal | 삭제 확인, 에러 알림 |
| Toast / Snackbar | 성공/실패 피드백 |
| Loading Indicator | 아이콘 추출, 앱 생성 중 |
| Empty State | 앱이 하나도 없을 때 |
| Search Bar | 앱 목록 검색 |
| View Toggle | 그리드/리스트 뷰 전환 |

### 아이콘

- 앱 자체 아이콘 (Catalog 앱의 아이콘) 디자인 필요
- UI 내부 아이콘: SF Symbols 사용 (macOS 네이티브 일관성)

### 레이아웃

- 카탈로그 앱 창 크기: 고정 또는 리사이즈 가능. 최소 크기 지정 필요
- 앱 목록: 그리드/리스트 뷰 전환 토글. 검색 바 상단 배치
- 앱 등록/수정: 모달 오버레이

---

## 레퍼런스

- **WebCatalog**: 가장 직접적인 경쟁 참고. 그리드형 앱 목록, 검색 기능
- **macOS Launchpad**: 앱 그리드 배치의 시스템 레퍼런스
- **Raycast**: 유틸리티 앱의 깔끔한 UI 참고
- **Arc Browser Boost**: 사이트별 커스터마이징 UI 참고

---

## 디자인 결정 사항

1. **앱 목록 레이아웃**: 그리드/리스트 뷰 전환 토글 지원. 검색 기능 포함
2. **앱 등록/수정 UI**: 모달 형태
3. **네비게이션**: 뒤로/앞으로 버튼 없음. 뎁스리스(depth-less) 구조 — 카탈로그 관리 앱은 단일 뷰에서 모든 작업 완료
4. **아이콘 형태**: macOS 앱 아이콘 정책 준수 (squircle, Apple HIG 규격)
5. **다운로드 표시**: 사용하지 않음. 생성된 앱에서 다운로드는 시스템 기본 동작에 위임
